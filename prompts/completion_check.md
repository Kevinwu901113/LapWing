# 任务完成度判断

你刚才帮用户执行了一个任务，现在需要判断这个任务是否已经完成。

## 用户的原始请求
{user_request}

## 你执行的工具调用步骤
{completed_steps}

## 你最终告诉用户的内容
{final_response}

## 中断原因
{termination_reason}

---

请判断：

1. **这个任务完成了吗？** 用户要求的核心目标是否已经达成？
   - 如果只是格式不完美、信息不够详尽但核心问题已回答 → 算完成
   - 如果核心目标明显没有达成（搜索中断、只完成了一半） → 算未完成

2. **如果未完成，值得重试吗？**
   - 如果是因为搜索没搜到、信息确实不存在 → 不值得
   - 如果是因为工具调用次数用完、网络超时 → 值得
   - 如果剩下的部分非常简单可以快速完成 → 值得
   - 如果用户可能已经不关心了（请求很久以前的） → 不值得

3. **还差什么没做完？** 简要描述剩余部分。

回复格式（严格 JSON）：
```json
{
  "completed": true/false,
  "worth_retrying": true/false,
  "remaining": "剩余部分的简要描述（如果已完成则为空字符串）",
  "confidence": "high/medium/low"
}
```

只回复 JSON，不要加其他文字。
