## 3. 核心模块清单

### src/adapters

- **base.py** — 抽象基类 `BaseAdapter` 和 `ChannelType` 枚举，定义所有消息通道适配器的接口。
- **desktop_adapter.py** — 将活跃的桌面端 WebSocket 连接封装为 `ChannelManager` 可路由的通道。
- **qq_adapter.py** — QQ 通道适配器，通过 WebSocket 与 NapCat（OneBot v11）通信。
- **qq_group_context.py** — 内存缓冲 `GroupContext`，跟踪每个 QQ 群聊的最近消息和状态。
- **qq_group_filter.py** — 基于 LLM 的参与决策器，判断 Lapwing 是否应回复群消息。

### src/agents

- **base.py** — `BaseAgent` 基类，实现所有 Agent 共享的通用工具循环。
- **coder.py** — Coder Agent，专注于在 `data/agent_workspace/` 沙箱中编写和执行代码。
- **registry.py** — `AgentRegistry`，运行时注册和查找命名 Agent 实例。
- **researcher.py** — Researcher Agent，专注于网络搜索、页面抓取和生成引用摘要。
- **team_lead.py** — Team Lead Agent，接收任务后委派给 Researcher/Coder 子 Agent 并汇总结果。
- **types.py** — Agent 共享数据类：`AgentSpec`、`AgentMessage`、`AgentResult`。

### src/api

- **event_bus.py** — `DesktopEventBus`，将进程内事件发布到桌面端 SSE 订阅队列。
- **model_routing.py** — `/api/model-routing` 路由，提供 LLM 提供商/槽位配置的 CRUD 操作。
- **server.py** — FastAPI 应用工厂 `create_app` 和 `LocalApiServer`，组装所有路由模块。

### src/api/routes

- **agents.py** — `GET /api/agents` 端点，列出所有已注册 Agent。
- **auth.py** — 认证端点：bootstrap token 交换、API session 创建、session 验证。
- **browser.py** — 浏览器子系统 REST 端点：标签页列表、页面状态、截图、导航控制。
- **chat_ws.py** — WebSocket `/ws/chat` 端点，接收桌面端消息并路由到 `Brain`。
- **data.py** — 会话、记忆文件、兴趣、任务流、提醒、日志等数据端点。
- **events_v2.py** — SSE 端点 `/api/v2/events`，支持 Last-Event-ID 断线重连。
- **identity.py** — `soul.md`、`constitution.md`、`voice.md` 的读写、快照历史和回滚端点。
- **logs.py** — `GET /api/logs/events` 旧版事件查询端点。
- **models_v2.py** — `/api/v2/models/` 模型路由配置 CRUD 端点。
- **notes_v2.py** — `/api/v2/notes/` 笔记文件树浏览、内容检索、关键词搜索、语义向量搜索。
- **permissions_v2.py** — `/api/v2/permissions/` 用户权限查看和运行时覆盖。
- **status_v2.py** — `/api/v2/status` 推断当前运行状态（idle/thinking/working）。
- **system.py** — 系统管理端点：进化、配置、心跳、人格、知识、延迟遥测。
- **system_v2.py** — `/api/v2/system/` CPU/内存/磁盘资源、运行时长、通道状态、事件日志。
- **tasks_v2.py** — `/api/v2/tasks/` 任务列表、详情和关联事件查询。

### src/app

- **container.py** — `AppContainer` 依赖注入根，管理完整应用生命周期（prepare → start → shutdown）。
- **task_view.py** — `TaskViewStore` 读模型，通过消费 `task.*` 事件重建统一任务视图。

### src/auth

- **models.py** — 认证相关数据类和类型别名。
- **openai_codex.py** — OpenAI Codex OAuth 提供商，实现 PKCE 授权流和 token 刷新。
- **resolver.py** — 将 `SecretRef` 对象（字面量、环境变量、shell 命令）解析为明文密钥。
- **service.py** — `AuthManager` 处理 API session cookie、bootstrap token、LLM 凭证管理。
- **storage.py** — 基于文件的认证 profile 存储，带文件锁和冷却/退避追踪。

### src/core

- **authority_gate.py** — 四级权限系统（IGNORE/GUEST/TRUSTED/OWNER），按操作检查授权。
- **background_review.py** — 异步后台回顾器，每 N 轮用独立 LLM 调用决定是否持久化对话内容。
- **brain.py** — 核心 `LapwingBrain`，编排对话流：上下文准备、系统提示组装、任务运行时调用。
- **browser_manager.py** — `BrowserManager`，Playwright/Chromium 自动化，带持久上下文和视觉管线。
- **channel_manager.py** — `ChannelManager`，按 `ChannelType` 注册适配器并路由出站消息。
- **checkpoint_manager.py** — 影子 git 仓库管理器，在变更操作前自动快照文件以支持回滚。
- **codex_oauth_client.py** — Codex Responses API 的 httpx 客户端，处理 SSE 流和 OAuth token 刷新。
- **consciousness.py** — `ConsciousnessEngine` 自主节拍循环，周期性向 Brain 注入内部提示。
- **credential_vault.py** — Fernet 加密存储，保存浏览器登录工具使用的网站凭据。
- **dispatcher.py** — 串行化 `Dispatcher`，通过单个异步锁接收所有状态变更事件并路由到处理器。
- **durable_scheduler.py** — 持久化提醒调度器，支持重复任务，基于 aiosqlite 分钟级轮询。
- **event_logger_v2.py** — 追加式 SQLite `EventLogger`（v2），存储所有系统事件的结构化记录。
- **group_filter.py** — QQ 群消息基于规则的预过滤器：分类为 enter/cache/discard。
- **heartbeat.py** — `HeartbeatEngine` 和 `HeartbeatAction` ABC，驱动 fast/slow/minute 节拍周期。
- **incident_filter.py** — 事件过滤器，快速丢弃不值得创建 incident 的工具失败。
- **incident_manager.py** — 管理 incident 的创建、去重（1 小时窗口）、每日限制、状态转换和自动升级。
- **knowledge_manager.py** — 基于文件的知识笔记管理器，`data/knowledge/` 下按主题键存储。
- **latency_monitor.py** — 滚动窗口 P95 延迟追踪器，带 SLO 告警。
- **llm_exceptions.py** — LLM 调用失败的类型化异常层次结构。
- **llm_protocols.py** — OpenAI 和 Anthropic 消息/工具调用格式之间的协议适配器。
- **llm_router.py** — `LLMRouter`，按 purpose slot 选择正确的 LLM 客户端和模型。
- **llm_types.py** — 核心 LLM 数据类型：`ToolCallRequest`、`ToolTurnResult`、`ModelOption`。
- **minimax_vlm.py** — MiniMax VLM 端点的 HTTP 客户端，用于浏览器视觉管线的图像理解。
- **model_config.py** — `ModelConfigManager`，将 LLM 路由配置持久化到 JSON 文件。
- **output_sanitizer.py** — 出站消息最后一道过滤，剥离内部标记（`[NEXT:...]`、`<think>` 等）。
- **pending_task.py** — `PendingTask` 数据类和文件存储，支持心跳驱动的中断任务恢复。
- **process_registry.py** — 后台进程注册表，追踪通过 `process_spawn` 启动的进程。
- **progress_reporter.py** — 循环内进度报告器，在长工具链中生成用户可见的进度更新。
- **prompt_builder.py** — 8 层系统提示组装器：soul → rules → time → memory → facts → vectors → summaries → voice。
- **prompt_loader.py** — 热加载 Markdown 提示加载器，从 `prompts/` 读取文件。
- **reasoning_tags.py** — `<think>` 块剥离、`[SPLIT]` 标记移除和多消息分割工具。
- **reminder_scheduler.py** — 事件驱动 `ReminderScheduler`，秒级精度触发提醒。
- **runtime_profiles.py** — `RuntimeProfile` 定义（chat_shell、coder_snippet 等），控制每个执行上下文可用的工具。
- **shell_policy.py** — `ShellRuntimePolicy`，从自然语言任务中提取执行约束并评估验证策略。
- **shell_types.py** — 共享 `VerificationStatus` 数据类。
- **soul_manager.py** — `SoulManager`，`soul.md` 版本管理：快照历史、差异生成、编辑冷却、回滚。
- **task_flow.py** — `TaskFlowManager`，多步骤任务流管理，带 JSON 检查点持久化和崩溃恢复。
- **task_model.py** — SQLite 支持的 `Task` 模型，带 `TaskBudget` 和状态生命周期。
- **task_runtime.py** — `TaskRuntime`，执行 LLM 工具循环：工具选择、执行、结果回传、循环检测、熔断器。
- **task_types.py** — TaskRuntime 数据类型：`LoopDetectionConfig`、`RuntimeDeps` 等。
- **time_utils.py** — 共享日期时间辅助函数。
- **trace_recorder.py** — 执行跟踪记录器，将每任务工具调用详情记录到日志文件。
- **trust_tagger.py** — `TrustTagger`，在注入 LLM 上下文前为传入内容添加 XML 信任级别标签。
- **verifier.py** — 统一执行后验证器：shell 任务完成、文件写入、Python 代码语法、工作区输出检查。
- **vital_guard.py** — `VitalGuard` 预执行护盾，阻止或要求备份后执行对受保护路径的写入。
- **vitals.py** — 系统生命体征：启动时间、运行时长、重启检测、CPU/内存/磁盘快照。

### src/guards
