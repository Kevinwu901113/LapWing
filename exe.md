# Lapwing 桌面端完整需求确认书

> 本文档汇总了所有经过确认的需求，作为后续 blueprint 的唯一输入。

---

## 一、定位

Lapwing 在 Windows 上的**唯一**本地程序。承载与 Lapwing 的全部交互：聊天（主力入口）、系统管理、环境感知。取代现有的 Tauri 半成品前端和网页前端，从零开始。

---

## 二、技术栈

| 层 | 技术 | 说明 |
|----|------|------|
| 框架 | Tauri v2 | 轻量（30-50MB 常驻），Rust 原生调 Windows API |
| 系统层 | Rust | 窗口监控、进程检测、剪贴板、文件监听、锁屏事件、开机自启、全局快捷键、系统托盘 |
| UI 框架 | React + TypeScript | 前端渲染 |
| 组件库 | shadcn/ui + Tailwind CSS + Radix UI | 拥有全部源码，深度定制 |
| 状态管理 | Zustand | 多 store 多数据源 |
| 路由 | React Router | 页面导航 |
| 图表 | Recharts | 仪表盘可视化 |
| 代码编辑器 | CodeMirror | 人格文件 Markdown 编辑 |
| 构建 | Vite | 快速开发构建 |
| 自动更新 | Tauri Updater 插件 | 内置自动更新 |

## 三、视觉风格

**风格 E：深空面板**

- 深色背景（#0c0c0c 区间），非纯黑
- 侧边栏带微弱蓝色渐变
- Lapwing 头像使用蓝白渐变（呼应白发蓝眸）
- 消息气泡半透明，极细边框
- 文字颜色分层：主文字 #c8cdd8，次要信息 rgba(255,255,255,0.3)
- 发送按钮使用蓝白渐变
- 整体克制、安静、不花哨

---

## 四、窗口行为

- **关闭按钮**：最小化到系统托盘，不退出
- **托盘图标**：右键菜单（显示/隐藏、退出）
- **全局快捷键**：一键呼出/隐藏窗口
- **开机自启**：注册 Windows 启动项，启动后自动连接服务器
- **退出**：仅通过托盘右键"退出"完全关闭

## 五、页面结构（8 页）

### 1. 对话
- 与 Lapwing 的主聊天窗口，**主力聊天入口**
- 消息气泡、图片、[SPLIT] 多条消息展示
- 与 QQ 共享同一对话流（同一 chat_id），打开即可看到完整历史（含 QQ 上发的）
- 非实时同步——QQ 上发的消息不会实时推送到桌面端，但打开后能看到历史
- Session 自然切换，用户无感

### 2. 任务中心
- Agent 执行观察，独立页面（非弹出面板）
- 看到 Lapwing 拆分任务、分配给 Agent 的过程
- 各 Agent 的执行进度和输出
- 任务流进度卡片
- 纯观察模式，不需要用户介入
- 能取消运行中的任务

### 3. 仪表盘
- 系统监控全览，信息尽可能丰富
- 服务器资源（CPU/RAM/Disk）
- Lapwing 运行时长 / 上次重启时间
- 今日对话轮数
- 各通道连接状态（QQ/桌面端）+ 重连能力
- 心跳引擎状态（上次执行什么 action、下次什么时候）
- 最近的工具调用记录
- API 调用量 / token 消耗统计
- 记忆系统健康度
- 最近一次自省/进化的时间和摘要
- 今日主动消息次数
- 日历视图 + 提醒列表（日程管理融入仪表盘）
- 更多待设计阶段穷举

### 4. 环境感知
- 应用使用时间线（今天用了什么应用、各用了多久）
- 当前前台应用状态
- 免打扰模式状态（是否在游戏中）
- 数据从本地 SQLite 读取，不走服务器
- 同时作为展示给用户的仪表盘，也喂给 Lapwing 的 SenseContext

### 5. 记忆
- 统一管理所有记忆类内容，用 tab 分类：
  - 对话记忆（KEVIN.md / SELF.md / 对话摘要）
  - 用户画像（SQLite facts）
  - 自省日志（journal/）
  - 知识笔记（knowledge/）
  - 兴趣图谱（interests.md）
- 支持浏览、搜索、编辑、删除（完整 CRUD）

### 6. 人格
- soul.md / voice.md / examples.md / capabilities.md 等文件的 Markdown 编辑器
- 进化历史查看（changelog.md），diff 对比
- 手动触发进化
- 技能管理：已安装技能列表、经验技能执行统计、启用/禁用

### 7. 模型路由
- Provider 管理（增删改）：base_url、api_type、API key 配置
- Slot 分配：哪个模型用于哪个语义 slot
- 热重载
- 模型测试：发一条测试消息看回复质量

### 8. 设置
- 通用配置（心跳间隔、消息合并时间等）
- 快捷键设置
- 提示音设置（自定义短音效）
- 日志流（实时日志，保留但不突出）
- 通道参数（QQ WebSocket URL 等）
- Feature Flag 开关
- 关于 / 版本信息

## 六、系统感知能力

所有能力均不触发 VAC / PAC / 5EAC 反作弊。

| 能力 | 实现方式 | 数据存储 |
|------|----------|----------|
| 前台窗口标题 + 进程名 | `GetForegroundWindow` + `GetWindowText`，每秒轮询 | 本地 SQLite |
| 应用使用时长统计 | 基于窗口标题变化聚合 | 本地 SQLite |
| 开机/关机检测 | Windows 启动项 + `WM_QUERYENDSESSION` | 事件推送服务器 |
| 锁屏/解锁检测 | `WTSRegisterSessionNotification` | 事件推送服务器 |
| 游戏检测/免打扰 | 进程名匹配 + 全屏独占检测 | 本地状态 |
| 剪贴板内容 | 剪贴板监听 API | 本地缓存，摘要推送 |
| 浏览器网页标题/URL | 浏览器扩展（远期） | 本地 SQLite |
| 文件系统变化 | `ReadDirectoryChangesW` | 本地缓存，摘要推送 |

**数据流策略**：原始数据存本地 SQLite，每隔几分钟聚合成环境摘要（几十 token），推送给服务器 SenseContext。环境感知页面直接读本地数据库。

**游戏静默模式**：检测到游戏进程（cs2.exe 等）或全屏独占时：
- 暂停所有系统 API 采集
- 隐藏到托盘
- 只保留 WebSocket 心跳连接
- 提醒系统不关（用系统通知弹窗）
- 游戏退出后自动恢复，Lapwing 可主动回应（"打完了？怎么样"）

**游戏学习**：遇到未知的全屏进程时，Lapwing 可以询问是否是游戏，确认后自动加入免打扰列表。

## 七、通知系统

- **应用内气泡**：窗口在后台时，托盘图标闪烁/红点
- **自定义提示音**：清脆短音效（叮一声）
- 不使用 Windows 系统通知弹窗（除游戏中的提醒外）

## 八、通道关系

| 通道 | 状态 | 用途 |
|------|------|------|
| 桌面端 | 主力 | 聊天 + 管理 + 感知 |
| QQ | 保留 | 私聊 + 群聊（群聊不在桌面端展示） |
| Telegram | 下线 | 已由桌面端取代 |

- 桌面端和 QQ 共享同一对话流
- QQ 群聊完全留在 QQ 端
- 手机端为远期规划

## 九、核心设计原则

### "主动"是最核心的特性

Lapwing 区别于所有其他 AI 产品的本质。系统感知不是一个附加功能，而是让她能对用户状态变化做出自然反应的基础设施：
- 游戏结束后主动问候
- 连续工作太久主动提醒
- 深夜未锁屏关心
- 刚开机打招呼
- 切到特定项目时主动关联记忆

### 其他原则（继承自愿景文档）
- 她是人不是工具
- 文件是 source of truth
- 人格通过 prompt 而非代码定义
- Diff-based 进化，宪法保护核心
