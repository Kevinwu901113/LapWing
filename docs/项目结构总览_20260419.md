## 1. Python 后端：`src/`

### 1.1 `src/app/` — 应用装配

| 文件 | 作用 |
|------|------|
| `container.py` | DI 根容器：装配 Brain / MainLoop / EventQueue / ChannelManager / Dispatcher / DurableScheduler / TrajectoryStore / CommitmentStore / Attention / LocalApiServer |
| `task_view.py` | 任务视图（给 `/api/v2/life/timeline`） |

### 1.2 `src/core/` — 核心运行时（44 个文件）

**对话主路径**
- `brain.py` — `LapwingBrain`，所有入口最终进这里
- `task_runtime.py` — 工具循环（LLM ↔ tool，带 loop detection / error budget）
- `llm_router.py` — 按槽位 (chat/tool/heartbeat/browser_vision) 路由到不同模型
- `llm_protocols.py` — OpenAI / Anthropic / NIM 协议适配
- `llm_types.py` / `llm_exceptions.py` — LLM 数据类型和异常
- `task_types.py` / `task_model.py` — 任务运行时数据结构
- `reasoning_tags.py` — 处理 `<internal_thinking>` 等内部标签
- `output_sanitizer.py` — 输出最终过滤

**单消费者事件循环（Blueprint v2.0）**
- `main_loop.py` — 单消费者事件循环：MessageEvent / InnerTickEvent / SystemEvent
- `event_queue.py` — 带优先级 peek 的 asyncio 队列
- `events.py` — 事件类型定义
- `inner_tick_scheduler.py` — 自发意识节拍（空闲时 backoff，OWNER 打断）
- `attention.py` — 当前聚焦状态（对话/动作/空闲）
- `dispatcher.py` — 内存 pub/sub（过渡态，逐步被 mutation_log 取代）
- `trajectory_store.py` — 跨频道只追加行为时间线
- `commitments.py` — 承诺存储（commit / fulfill / abandon + deadline）
- `durable_scheduler.py` — 持久化提醒（reminders_v2 表，notify/agent 模式）
- `system_send.py` — 框架级用户消息（mirrors tell_user trajectory recording）

**状态视图 / 提示词组装**
- `state_view.py` — 不可变快照类型
- `state_view_builder.py` — 从各 store 拼出 StateView
- `state_serializer.py` — 纯函数：StateView → prompt bytes
- `identity_file_manager.py` — soul.md / constitution.md I/O + snapshot/rollback
- `soul_manager.py` — soul.md 版本控制（24h cooldown）
- `prompt_loader.py` — prompts/*.md 热重载 + 缓存
- `phase0.py` — 极简测试 harness（PHASE0_MODE="A"/"B"）

**安全与权限**
- `authority_gate.py` — IGNORE(0) / GUEST(1) / TRUSTED(2) / OWNER(3) 四级
- `vital_guard.py` — Shell/文件写入前盾（PASS / VERIFY_FIRST / BLOCK）
- `shell_policy.py` — 目录 / sudo / 扩展名约束
- `shell_types.py` — Shell 数据类型
- `verifier.py` — 代码 / workspace 验证
- `credential_vault.py` — Fernet 加密凭证库

**感知 / 环境**
- `vitals.py` — boot time / uptime / 系统快照（CPU/memory/disk via psutil）
- `maintenance_timer.py` — 每日 3AM 语义蒸馏
- `runtime_profiles.py` — 按场景定义可用工具集
- `model_config.py` — 运行时模型绑定（persisted to data/config/model_routing.json）
- `group_filter.py` — QQ 群消息 LLM 过滤
- `time_utils.py` — 日期/时间工具
- `trust_tagger.py` — 给来源消息打信任标签
- `minimax_vlm.py` — MiniMax VLM 调用封装
- `browser_manager.py` — Playwright 浏览器自动化 + 内置 BrowserGuard
- `channel_manager.py` — 多通道发送路由
- `codex_oauth_client.py` — OpenAI/Codex PKCE 流程
